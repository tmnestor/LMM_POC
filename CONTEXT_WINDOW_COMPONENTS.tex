\section*{Executive Summary}

This document provides a complete breakdown of how the context window is consumed in vision-language model inference. The analysis shows that the current implementation uses only \textbf{8-14\% of available context}, providing significant safety margins for production deployment.

\textbf{Key Findings}:
\begin{itemize}[leftmargin=*]
    \item \textbf{InternVL3-8B}: Uses ~4,724 tokens of 32,768 (14.4\% utilization)
    \item \textbf{Llama-3.2-11B}: Uses ~10,796 tokens of 128,000 (8.4\% utilization)
    \item \textbf{Safety margin}: 3.8x to 9.9x headroom in worst-case scenarios
    \item \textbf{Stateless design}: No conversation history accumulation keeps usage constant
\end{itemize}

\section{Context Window Components Overview}

The context window is consumed by \textbf{five main components} during inference:

\subsection{Component Summary}

\begin{enumerate}[leftmargin=*]
    \item \textbf{System Prompt / Instructions (Optional)}
    \begin{itemize}
        \item Pre-instructions to guide model behavior
        \item \textbf{Current implementation}: Not used (0 tokens)
        \item \textbf{Typical range}: 0-500 tokens
    \end{itemize}

    \item \textbf{User Prompt (Extraction Instructions)}
    \begin{itemize}
        \item Document-aware extraction prompt
        \item \textbf{Current implementation}: 500-800 tokens
        \item Varies by document type (invoice, receipt, bank statement)
    \end{itemize}

    \item \textbf{Image(s) (Visual Tokens)}
    \begin{itemize}
        \item Encoded representation of document image
        \item \textbf{InternVL3-8B}: 1,536-5,120 tokens (6-20 tiles)
        \item \textbf{Llama-3.2-11B}: 4,000-8,000 tokens (high-res documents)
    \end{itemize}

    \item \textbf{Conversation History (Optional)}
    \begin{itemize}
        \item Previous turns in multi-turn conversations
        \item \textbf{Current implementation}: 0 tokens (stateless processing)
        \item \textbf{If enabled}: Would grow with each turn
    \end{itemize}

    \item \textbf{Model Response (Generated Output)}
    \begin{itemize}
        \item The extracted data being generated
        \item \textbf{InternVL3}: 1,024 tokens max (max\_new\_tokens)
        \item \textbf{Llama}: 4,096 tokens max (max\_new\_tokens)
    \end{itemize}
\end{enumerate}

\subsection{Image Token Consumption}

\subsubsection{InternVL3 Image Encoding}

\textbf{Architecture}: Dynamic tiling with pixel unshuffle

\textbf{Process}:
\begin{enumerate}[leftmargin=*]
    \item Image divided into 448×448 pixel tiles
    \item Pixel unshuffle reduces tokens to 1/4 of original
    \item Each tile = \textbf{256 tokens}
    \item Variable number of tiles based on aspect ratio
\end{enumerate}

\textbf{Configuration}:
\begin{itemize}[leftmargin=*]
    \item \textbf{Max tiles (8B model)}: 20 tiles (configurable)
    \item \textbf{Worst case}: 20 × 256 = 5,120 tokens
    \item \textbf{Typical document}: 6-12 tiles = 1,536-3,072 tokens
\end{itemize}

\textbf{Official Documentation}:
\begin{quote}
\textit{"InternVL3 applies a pixel unshuffle operation that reduces visual tokens to one-quarter of the original value. Each 448×448 image tile is represented with 256 visual tokens."}
\end{quote}

\textbf{Reference}: \url{https://arxiv.org/html/2504.10479v1}

\subsubsection{Llama 3.2 Vision Image Encoding}

\textbf{Architecture}: Fixed encoding based on resolution

\textbf{Official Measurements}:
\begin{itemize}[leftmargin=*]
    \item \textbf{512×512 image}: ~1,610 tokens
    \item \textbf{Maximum resolution}: 1120×1120 pixels
    \item \textbf{Scaling}: Approximately linear with pixel count
\end{itemize}

\textbf{Scaling Formula}:
\begin{equation}
\text{tokens} = \frac{(\text{width} \times \text{height}) \times 1610}{512 \times 512}
\end{equation}

\textbf{Official Documentation}:
\begin{quote}
\textit{"Llama 3.2 VLMs support long context lengths of up to 128K text tokens as well as a single image input at a resolution of 1120 x 1120 pixels. A 512 x 512 image is converted to about 1,610 tokens."}
\end{quote}

\textbf{Reference}: \url{https://docs.oracle.com/en-us/iaas/Content/generative-ai/meta-llama-3-2-11b.htm}

\section{Key Takeaways}

\begin{enumerate}[leftmargin=*]
    \item \textbf{Extremely Conservative Settings}

    Both models use only 8-14\% of available context, providing massive safety margins:
    \begin{itemize}
        \item InternVL3-8B: 85.6\% unused
        \item Llama-3.2-11B: 91.6\% unused
    \end{itemize}

    \item \textbf{Images Are Relatively Cheap}

    Despite being multimodal, images consume a small portion of context:
    \begin{itemize}
        \item InternVL3: 4.7\% - 15.6\% for images
        \item Llama: 3.1\% - 6.25\% for images
    \end{itemize}

    \item \textbf{Stateless Design Is Optimal}

    Zero tokens allocated to conversation history:
    \begin{itemize}
        \item Keeps usage constant across batch
        \item Prevents context accumulation
        \item Maintains predictable behavior
        \item Perfect for independent document processing
    \end{itemize}

    \item \textbf{Significant Growth Potential}

    With 3.8x to 9.9x headroom, you could:
    \begin{itemize}
        \item Increase response length 5-10x
        \item Add conversation history (3-10 turns)
        \item Process multiple images per request
        \item Add detailed system prompts
        \item Implement multi-turn refinement
    \end{itemize}

    \item \textbf{Provider Compatibility}

    Current settings work on most platforms:
    \begin{itemize}
        \item \cmark{} Self-hosted: Full support
        \item \cmark{} AWS Bedrock: Full support
        \item \cmark{} Oracle Cloud: Full support
        \item ⚠ Azure: Llama exceeds 8K limit
    \end{itemize}

    \item \textbf{Production-Ready Configuration}

    Your settings align with official recommendations:
    \begin{itemize}
        \item InternVL3: 1,024 tokens (matches official default)
        \item Llama: 4,096 tokens (within recommended 2,048-4,096 range)
        \item Both provide excellent safety margins
        \item Well-suited for production deployment
    \end{itemize}
\end{enumerate}

\subsection{Future Enhancements (If Needed)}

\begin{enumerate}[leftmargin=*]
    \item \textbf{Increase Response Length} (Low Priority)
    \begin{itemize}
        \item Current: 1,024 / 4,096 tokens
        \item Could increase: 2,048 / 8,192 tokens
        \item Rationale: Handle more complex documents
    \end{itemize}

    \item \textbf{Add System Prompt} (Optional)
    \begin{itemize}
        \item Cost: ~200-500 tokens
        \item Benefit: Consistent model behavior
        \item Trade-off: Minimal impact on context
    \end{itemize}

    \item \textbf{Multi-Turn Refinement} (Advanced)
    \begin{itemize}
        \item Enable conversation history
        \item Allow follow-up corrections
        \item Cost: ~4,724 / 10,796 tokens per turn
        \item Rationale: Iterative extraction quality improvement
    \end{itemize}

    \item \textbf{Multiple Images} (Future Feature)
    \begin{itemize}
        \item Process multi-page documents
        \item Cost: Additional ~3,000 / 6,000 tokens per image
        \item Limit: 5-9 images (InternVL3), 14-19 images (Llama)
    \end{itemize}
\end{enumerate}
